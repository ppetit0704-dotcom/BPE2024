Base Permanente des Équipements

Présentation

Introduction

Le Tableau de bord BPE est une application Streamlit permettant d'explorer la Base Permanente des Équipements (BPE) publiée chaque année par l'INSEE. Elle s'adresse aussi bien aux élus et techniciens territoriaux souhaitant consulter les équipements de leur commune ou intercommunalité, qu'aux développeurs souhaitant l'adapter ou l'enrichir.

L'application offre :

  • La consultation des équipements par commune ou par EPCI
  • L'enrichissement contextuel via les zonages INSEE (Unités urbaines, Bassins de vie, Aires d'attraction des villes)
  • Un module de signalement d'anomalies BPE générant un PDF et un email pré-rempli
  • La récupération automatique du nom de l'EPCI via l'API geo.api.gouv.fr
Source des données : La BPE est produite annuellement par l'INSEE et mise à disposition en open data sur data.gouv.fr. Le millésime 2026 couvre l'ensemble du territoire métropolitain et des DOM.

Architecture générale

L'application suit une architecture modulaire : chaque fonctionnalité majeure est isolée dans son propre module Python, importé dans le point d'entrée app.py. La communication entre modules s'effectue exclusivement via st.session_state.

├── app.py                          # Point d'entrée principal
├── assets/
│   └── logo.png                    # Logo affiché dans le header
├── ui/
│   └── cards.py                    # Composants badge (badgeBlue, badgeGreen…)

Guide utilisateur

Import du fichier BPE

La BPE est distribuée par l'INSEE sous forme de fichier CSV zippé. Après décompression, le fichier se nomme généralement bpe26_ensemble.csv.

Téléchargement

Un lien de téléchargement direct depuis data.gouv.fr est disponible dans la barre latérale de l'application. Cliquer sur 🗺️ Télécharger les données depuis data.gouv.fr.

Import dans l'application

  1. Dans la sidebar, cliquer sur 📁 Import des données pour déplier le panneau
  2. Glisser-déposer le fichier CSV ou cliquer sur Browse files
  3. Une fois chargé, le sélecteur de territoire apparaît automatiquement
Le fichier BPE complet (France entière) pèse environ 150 Mo. Le chargement peut prendre quelques secondes selon la machine. Le DataFrame est mis en cache dans st.session_state["df"] pour éviter les rechargements.

Guide utilisateur

Sélection du territoire

Après import, le sélecteur de territoire s'affiche dans la sidebar sous 🔍 Sélectionnez votre territoire…

Mode Commune

  1. Choisir Commune dans le sélecteur de mode
  2. Saisir ou sélectionner le département, puis la commune
  3. Sélectionner optionnellement le domaine et le type d'équipement
  4. Les résultats s'affichent dans la zone principale, organisés par DOM → SDOM

Mode EPCI

  1. Choisir EPCI dans le sélecteur de mode
  2. Saisir le code EPCI (9 chiffres) ou le sélectionner dans la liste
  3. L'application interroge automatiquement geo.api.gouv.fr pour afficher le nom officiel de l'intercommunalité
  4. Le nombre de communes couvertes est indiqué dans le sous-titre
Astuce : En mode EPCI, les blocs de zonages INSEE (Unités urbaines, Bassins de vie, Aires d'attraction) listent les zonages distincts présents dans l'EPCI avec la liste des communes appartenant à chacun — particulièrement utile pour les intercommunalités à cheval sur plusieurs bassins.

Guide utilisateur

Signalement d'anomalie BPE

La BPE présente parfois des décalages par rapport à la réalité du terrain — équipements manquants, fermés ou mal géolocalisés. Le module de signalement permet de générer un courrier officiel de demande de mise à jour.

Accès au formulaire

Cliquer sur le bouton 📋 Signaler une anomalie BPE en bas de la barre latérale. Un bouton ← Retour à la consultation permet de revenir à la vue principale.

Remplissage

Le formulaire est organisé en 4 sections :

  1. Émetteur — Commune, code INSEE, contact, email, téléphone
  2. Équipement concerné — Nom, adresse, catégorie BPE, type, année de mise en service
  3. Description de l'anomalie — Description libre + justificatifs disponibles
  4. Destinataire — INSEE, INJEP/RES (équipements sportifs), ou DR INSEE régionale
Suggestion automatique du destinataire : Si la catégorie sélectionnée est Sport, le formulaire suggère automatiquement l'INJEP/RES comme destinataire, conformément au circuit de mise à jour officiel.

Sorties générées

  • PDF téléchargeable — Courrier officiel mis en page, prêt à être joint à un email ou imprimé
  • Email pré-rempli — Lien mailto: ouvrant la messagerie par défaut avec objet et corps structurés

Documentation technique

API geo.api.gouv.fr

L'application interroge l'API Géo d'Etalab pour résoudre le nom officiel d'un EPCI à partir de son code. Cette API est gratuite, sans clé d'authentification, et maintenue par la DINUM.

Endpoint utilisé

GET https://geo.api.gouv.fr/epcis/{code_epci}

# Exemple
GET https://geo.api.gouv.fr/epcis/200053767

# Réponse
{
  "code": "200053767",
  "nom": "Nîmes Métropole",
  "type": "CA",
  "populationTotale": 254582
}

Gestion du cache et des erreurs

La fonction get_nom_epci() dans app.py implémente :

  • Cache session — le résultat est stocké dans st.session_state[f"epci_nom_{code}"] pour éviter un appel API à chaque rerun Streamlit
  • Timeout — 3 secondes maximum pour ne pas bloquer l'interface
  • Fallback silencieux — en cas d'erreur réseau ou de code inconnu, le code brut est affiché sans message d'erreur
def get_nom_epci(code_epci: str) -> str:
    cache_key = f"epci_nom_{code_epci}"
    if cache_key in st.session_state:
        return st.session_state[cache_key]
    try:
        r = requests.get(
            f"https://geo.api.gouv.fr/epcis/{code_epci}",
            timeout=3
        )
        if r.status_code == 200:
            data = r.json()
            nom = data.get("nom", code_epci)
            type_epci = data.get("type", "")
            result = f"{nom} ({type_epci})" if type_epci else nom
        else:
            result = code_epci
    except Exception:
        result = code_epci
    st.session_state[cache_key] = result
    return result

Documentation technique

Distribution PyInstaller

L'application peut être packagée en exécutable Windows autonome via PyInstaller, sans nécessiter d'installation Python sur le poste utilisateur.

Mécanisme de résolution des chemins

En mode PyInstaller, les fichiers sont extraits dans un dossier temporaire _MEIPASS. Le bloc suivant en tête de app.py résout automatiquement le chemin correct :

_candidates = [
    getattr(sys, '_MEIPASS', None),
    os.path.dirname(os.path.abspath(__file__)),
    os.path.dirname(sys.executable),
    os.path.join(os.path.dirname(sys.executable), '_internal'),
]
for _p in _candidates:
    if _p and os.path.isdir(os.path.join(_p, 'ui')) and _p not in sys.path:
        sys.path.insert(0, _p)
        break

Arrêt de l'application

Le bouton 🚪 Quitter déclenche os._exit(0) via un Timer(2) pour laisser le temps à Streamlit de rendre la page de sortie avant de tuer le processus.

Attention : os._exit(0) est une sortie forcée qui ne déclenche pas les finaliseurs Python. C'est intentionnel pour couper proprement le serveur Streamlit intégré.

Questions fréquentes

Le nom de l'EPCI ne s'affiche pas

L'application a besoin d'une connexion Internet pour interroger l'API geo.api.gouv.fr. En cas d'indisponibilité, le code EPCI brut est affiché à la place — le reste de l'application fonctionne normalement.

Les zonages INSEE ne s'affichent pas après chargement

Vérifier que les colonnes BV2022 et AAV2020 sont présentes dans votre fichier BPE. Ces colonnes sont disponibles dans la BPE à partir du millésime 2022. Pour les unités urbaines, la jointure se fait via CODGEO ↔ DEPCOM.

Le fichier BPE est trop volumineux à charger

Il est possible de pré-filtrer le fichier CSV sur un ou plusieurs départements avant import pour réduire le temps de chargement. L'application n'impose pas de limite de taille.

Le formulaire de signalement ne génère pas de PDF

Vérifier que tous les champs obligatoires (marqués *) sont remplis. Le bouton de génération PDF est désactivé tant que le formulaire est incomplet.

L'application plante au démarrage avec PyInstaller

Vérifier que le dossier ui/ et ses fichiers sont bien inclus dans le build PyInstaller via l'option --add-data. Le bloc de résolution de chemin en tête de app.py gère plusieurs stratégies de fallback.

Un équipement de mon territoire est absent de la BPE

Utiliser le module 📋 Signaler une anomalie BPE accessible depuis la barre latérale. Pour les équipements sportifs, le signalement doit être adressé à l'INJEP via le Recensement des Équipements Sportifs (RES) sur res.injep.fr.
